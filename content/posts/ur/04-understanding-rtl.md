## تعارف

دائیں سے بائیں (RTL) متن کی سپورٹ اکثر ویب ڈویلپمنٹ میں بعد کی سوچ کے طور پر سمجھی جاتی ہے — ایک خصوصیت جو "کبھی" شامل کی جائے جب بین الاقوامی مارکیٹس اہم ہو جائیں۔ یہ نقطہ نظر تکلیف دہ ریٹروفٹنگ، ٹوٹی ہوئی لے آؤٹس، اور مایوس صارفین کا باعث بنتا ہے۔

حقیقت یہ ہے کہ RTL زبانیں دنیا بھر میں 500 ملین سے زیادہ لوگ بولتے ہیں۔ صرف عربی عالمی سطح پر پانچویں سب سے زیادہ بولی جانے والی زبان ہے۔ فارسی، عبرانی، اردو، اور پشتو میں سے ہر ایک کے لاکھوں بولنے والے ہیں جو ایسے انٹرفیس کے مستحق ہیں جو ان کے لیے قدرتی طور پر کام کریں۔

## RTL کیا ہے؟

### بنیادی باتیں

RTL (دائیں سے بائیں) ان تحریری نظاموں سے مراد ہے جہاں متن دائیں سے بائیں لکھا اور پڑھا جاتا ہے۔ اہم RTL رسم الخط ہیں:

| رسم الخط | زبانیں | بولنے والے |
|---------|--------|-----------|
| عربی | عربی، فارسی، اردو، پشتو، کردی، اویغور | ~400+ ملین |
| عبرانی | عبرانی، یدش | ~9 ملین |
| سریانی | سریانی، آرامی | ~400,000 |
| ثانا | دیویہی (مالدیپ) | ~350,000 |

### بالکل کیا دائیں سے بائیں بہتا ہے؟

RTL رسم الخط میں، درج ذیل دائیں سے بائیں بہتے ہیں:
- الفاظ کے اندر حروف
- جملوں میں الفاظ
- لائنوں میں پڑھنے کا ترتیب
- عمومی بصری اسکیننگ

تاہم، کچھ عناصر LTR رہ سکتے ہیں:
- سرایت شدہ لاطینی متن یا نمبر (کبھی کبھی)
- ریاضیاتی علامات
- پروگرامنگ کوڈ

اس اختلاط کو **دو طرفہ متن** یا "BiDi" کہا جاتا ہے، اور اس کے اپنے پیچیدہ قواعد ہیں۔

## RTL بمقابلہ LTR: اہم فرق

### بصری لے آؤٹ

جب صفحہ RTL کے لیے الٹا ہو:

```
LTR لے آؤٹ:                    RTL لے آؤٹ:
┌─────────────────────┐       ┌─────────────────────┐
│ لوگو   [نیو] [نیو]  │       │ [نیو] [نیو]   لوگو │
├─────────────────────┤       ├─────────────────────┤
│ سائیڈبار │  مواد    │       │    مواد  │ سائیڈبار │
└─────────────────────┘       └─────────────────────┘
```

### کیا الٹا ہونا چاہیے

وہ عناصر جو عام طور پر RTL میں الٹ جاتے ہیں:
- نیویگیشن ترتیب
- سائیڈبار پوزیشنز
- سمت بتانے والے آئیکنز (تیر، شیورنز)
- پروگریس انڈیکیٹرز
- سوائپ کی سمتیں
- متن کی سیدھ (بائیں سیدھ دائیں سیدھ بن جاتی ہے)
- مارجن اور پیڈنگ (بائیں دائیں بن جاتا ہے)

### کیا الٹا نہیں ہونا چاہیے

کچھ عناصر بغیر تبدیلی کے رہنے چاہئیں:
- **آئیکنز**: پلے بٹن، چیک مارکس، زیادہ تر غیر سمتی آئیکنز
- **تصاویر**: فوٹوز، مثالیں (جب تک ثقافتی طور پر نامناسب نہ ہوں)
- **فون نمبر**: ہمیشہ معیاری ترتیب میں ڈسپلے ہوتے ہیں
- **ویڈیو کنٹرولز**: پلے/پاز معیاری رہتے ہیں

## HTML/CSS میں RTL نافذ کرنا

### dir خصوصیت

RTL سپورٹ کی بنیاد HTML `dir` خصوصیت ہے:

```html
<!-- دستاویز کی سطح پر سمت مقرر کریں -->
<html lang="ar" dir="rtl">

<!-- یا مخصوص عناصر پر -->
<div dir="rtl">
  عربی میں مواد
</div>
```

### CSS منطقی خصوصیات

جدید CSS **منطقی خصوصیات** پیش کرتا ہے جو خود بخود تحریر کی سمت کے مطابق ڈھل جاتی ہیں:

```css
/* جسمانی خصوصیات (ڈھلتی نہیں) */
.پرانا-طریقہ {
  margin-left: 20px;
  padding-right: 10px;
  text-align: left;
}

/* منطقی خصوصیات (سمت کے مطابق ڈھلتی ہیں) */
.نیا-طریقہ {
  margin-inline-start: 20px;
  padding-inline-end: 10px;
  text-align: start;
}
```

### Flexbox اور Grid

Flexbox اور CSS Grid قدرتی طور پر RTL کی سپورٹ کرتے ہیں:

```css
.container {
  display: flex;
  flex-direction: row; /* RTL میں خود بخود الٹ جاتا ہے */
}
```

کوئی اضافی CSS کی ضرورت نہیں — بس کنٹینر پر `dir="rtl"` مقرر کریں۔

## اہم نکات

۱. **RTL کی جلد منصوبہ بندی کریں**: RTL سپورٹ کو بعد میں شامل کرنا شروع سے بنانے سے کہیں مشکل ہے۔

۲. **منطقی خصوصیات استعمال کریں**: CSS منطقی خصوصیات (`inline-start`, `inline-end`) خود بخود تحریر کی سمت کے مطابق ڈھل جاتی ہیں۔

۳. **Flexbox/Grid کا فائدہ اٹھائیں**: جدید لے آؤٹ سسٹمز سمت کی تبدیلیوں کو خود بخود ہینڈل کرتے ہیں۔

۴. **سب کچھ الٹا نہیں ہوتا**: کچھ عناصر (فوٹوز، ویڈیوز، کچھ آئیکنز) RTL میں نہیں پلٹنے چاہئیں۔

۵. **حقیقی صارفین کے ساتھ ٹیسٹ کریں**: مقامی RTL بولنے والے ایسے مسائل پکڑیں گے جو خودکار ٹیسٹنگ سے چھوٹ جاتے ہیں۔
