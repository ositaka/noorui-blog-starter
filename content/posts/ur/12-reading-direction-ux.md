## تعارف

جب ہم کوئی ویب سائٹ یا ایپ کھولتے ہیں، ہماری آنکھیں فطری طور پر ایک مخصوص راستے پر چلتی ہیں۔ انگریزی بولنے والے اوپر بائیں سے شروع کرتے ہیں؛ عربی یا اردو بولنے والے اوپر دائیں سے۔ یہ فرق صرف متن کی سمت کا نہیں — یہ پورے صارف کے تجربے کو متاثر کرتا ہے۔

سمجھنا کہ پڑھنے کی سمت UX کو کیسے متاثر کرتی ہے، کثیر لسانی مصنوعات بنانے کے لیے ضروری ہے۔

## آنکھوں کی حرکت کے نمونے

### LTR صارفین: F-پیٹرن

انگریزی اور دیگر LTR زبانوں کے صارفین F شکل میں اسکین کرتے ہیں:

```
LTR F-پیٹرن:
┌─────────────────────────────┐
│ ████████████████            │ ← پہلی افقی حرکت
│ ██████████                  │ ← دوسری افقی حرکت
│ █                           │
│ █                           │ ← عمودی اسکین
│ █                           │
└─────────────────────────────┘
```

### RTL صارفین: الٹا F-پیٹرن

عربی اور اردو صارفین آئینے کی طرح F اسکین کرتے ہیں:

```
RTL الٹا F-پیٹرن:
┌─────────────────────────────┐
│            ████████████████ │ ← پہلی افقی حرکت
│                  ██████████ │ ← دوسری افقی حرکت
│                           █ │
│                           █ │ ← عمودی اسکین
│                           █ │
└─────────────────────────────┘
```

## نیویگیشن کی توقعات

### مینو پوزیشن

صارفین مینو کو اپنی پڑھنے کی سمت کے آغاز میں تلاش کرتے ہیں:

```
LTR توقع:
┌───────────────────────────────┐
│ [مینو]  [لوگو]      [سرچ] [لاگ ان] │
└───────────────────────────────┘

RTL توقع:
┌───────────────────────────────┐
│ [لاگ ان] [سرچ]      [لوگو]  [مینو] │
└───────────────────────────────┘
```

### سائیڈبار

```
LTR:                    RTL:
┌────┬──────────┐      ┌──────────┬────┐
│سائڈ│          │      │          │سائڈ│
│بار │  مواد    │      │  مواد    │بار │
│    │          │      │          │    │
└────┴──────────┘      └──────────┴────┘
```

## بٹنز اور ایکشنز

### پرائمری ایکشن کی جگہ

پرائمری ایکشن بٹن عام طور پر "آگے" کی سمت میں ہوتا ہے:

```css
/* LTR: پرائمری دائیں طرف */
.button-group-ltr {
  display: flex;
  justify-content: flex-end;
}

/* RTL: پرائمری بائیں طرف */
.button-group-rtl {
  display: flex;
  justify-content: flex-start;
}

/* بہتر: منطقی خصوصیات */
.button-group {
  display: flex;
  justify-content: flex-end; /* RTL میں خود بخود الٹ */
}
```

### فارم بٹنز

```html
<!-- LTR ترتیب -->
<div class="form-actions">
  <button class="secondary">منسوخ</button>
  <button class="primary">محفوظ کریں</button>
</div>

<!-- RTL میں ترتیب خود بخود الٹ جائے -->
```

## پروگریس اور بہاؤ

### پروگریس بار

```
LTR پروگریس:
[████████░░░░░░░░] 50%
← شروع              آخر →

RTL پروگریس:
[░░░░░░░░████████] 50%
→ آخر              شروع ←
```

### اسٹیپر (Stepper)

```
LTR اسٹیپر:
(1) ─── (2) ─── (3) ─── (4)
شروع                    آخر

RTL اسٹیپر:
(4) ─── (3) ─── (2) ─── (1)
آخر                    شروع
```

```css
/* CSS میں اسٹیپر */
.stepper {
  display: flex;
  flex-direction: row;
  /* dir="rtl" میں خود بخود الٹ */
}
```

## کیروسل اور گیلری

### سوائپ کی سمت

```
LTR: پچھلا ← | → اگلا
RTL: اگلا ← | → پچھلا
```

```javascript
// سوائپ کی سمت کے لیے
const isRTL = document.dir === 'rtl';

carousel.on('swipe', (direction) => {
  if (direction === 'left') {
    isRTL ? showPrevious() : showNext();
  } else {
    isRTL ? showNext() : showPrevious();
  }
});
```

### نیویگیشن تیر

```html
<div class="carousel-controls">
  <button class="prev" aria-label="پچھلا">
    <!-- RTL میں → ، LTR میں ← -->
  </button>
  <button class="next" aria-label="اگلا">
    <!-- RTL میں ← ، LTR میں → -->
  </button>
</div>
```

## فارم ڈیزائن

### لیبل اور ان پٹ

```
LTR فارم:
نام: [_______________]
ای میل: [_______________]

RTL فارم:
[_______________] :نام
[_______________] :ای میل
```

```css
.form-group {
  display: flex;
  flex-direction: row;
  align-items: center;
}

.form-label {
  margin-inline-end: 8px;
}
```

### ان پٹ آئیکنز

```
LTR: [🔍|_______________]
RTL: [_______________|🔍]
```

```css
.search-input {
  padding-inline-start: 40px; /* آئیکن کی جگہ */
}

.search-icon {
  position: absolute;
  inset-inline-start: 12px;
}
```

## ٹیبلز اور ڈیٹا

### ٹیبل ہیڈرز

```
LTR ٹیبل:
┌──────────┬────────┬────────┐
│ نام      │ عمر    │ شہر    │
├──────────┼────────┼────────┤
│ احمد     │ 25     │ لاہور  │
└──────────┴────────┴────────┘

RTL ٹیبل:
┌────────┬────────┬──────────┐
│ شہر    │ عمر    │ نام      │
├────────┼────────┼──────────┤
│ لاہور  │ 25     │ احمد     │
└────────┴────────┴──────────┘
```

### نمبر کالمز

نمبروں والے کالمز RTL میں بھی بائیں سیدھ رکھیں:

```css
.table-number {
  text-align: end; /* RTL میں بائیں، LTR میں دائیں */
}
```

## موبائل UX

### Thumb Zone

```
LTR موبائل:           RTL موبائل:
┌─────────────┐       ┌─────────────┐
│             │       │             │
│             │       │             │
│ [آسان رسائی]│       │[آسان رسائی] │
│      ───────┘       └───────      │
│      دائیں انگوٹھا    │      بائیں انگوٹھا  │
```

### نیچے نیویگیشن

```html
<!-- RTL میں آئٹمز کی ترتیب الٹ جائے -->
<nav class="bottom-nav" dir="rtl">
  <a href="/">گھر</a>
  <a href="/search">تلاش</a>
  <a href="/cart">کارٹ</a>
  <a href="/profile">پروفائل</a>
</nav>
```

## رنگ اور ثقافت

### سمت اور معنی

بعض رنگ سمتوں سے جڑے ہیں:

- **سبز = آگے بڑھنا**: LTR میں دائیں، RTL میں بائیں
- **سرخ = پیچھے جانا**: LTR میں بائیں، RTL میں دائیں

```css
/* پروگریس انڈیکیٹر */
.step-completed {
  background: green;
}

.step-current {
  background: blue;
}

.step-upcoming {
  background: gray;
}
```

## ٹیسٹنگ چیک لسٹ

### RTL UX ٹیسٹنگ

```markdown
□ مینو صحیح طرف ہے
□ سائیڈبار صحیح پوزیشن میں ہے
□ پروگریس بار صحیح سمت بھرتا ہے
□ سوائپ صحیح سمت کام کرتا ہے
□ فارم لیبلز صحیح سیدھ میں ہیں
□ بٹنز کی ترتیب درست ہے
□ تیر صحیح سمت اشارہ کرتے ہیں
□ ٹیبلز صحیح طریقے سے پلٹی ہیں
```

## اہم نکات

۱. **آنکھوں کا بہاؤ الٹ ہے**: RTL صارفین دائیں سے بائیں اسکین کرتے ہیں۔

۲. **"آگے" کا مطلب بدلتا ہے**: LTR میں آگے = دائیں، RTL میں آگے = بائیں۔

۳. **Flexbox خود بخود کام کرتا ہے**: `dir` خصوصیت کا احترام کرتا ہے۔

۴. **سوائپ کی سمت چیک کریں**: کیروسل اور گیلری میں سمت درست کریں۔

۵. **حقیقی صارفین سے ٹیسٹ کروائیں**: مقامی RTL صارفین بہترین فیڈبیک دیتے ہیں۔
