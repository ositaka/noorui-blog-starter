## مقدمة

غالباً ما يُعامل دعم النص من اليمين إلى اليسار (RTL) كفكرة لاحقة في تطوير الويب — ميزة للإضافة "يوماً ما" عندما تصبح الأسواق الدولية مهمة. يؤدي هذا النهج إلى تعديلات مؤلمة، وتخطيطات معطلة، ومستخدمين محبطين.

الحقيقة هي أن لغات RTL يتحدثها أكثر من ٥٠٠ مليون شخص حول العالم. العربية وحدها هي خامس أكثر اللغات تحدثاً عالمياً. الفارسية والعبرية والأردية والبشتونية لكل منها ملايين المتحدثين الذين يستحقون واجهات تعمل بشكل طبيعي لهم.

يغطي هذا الدليل كل ما تحتاج معرفته عن دعم RTL: ماذا يعني، كيف يعمل، وكيفية تنفيذه بشكل صحيح.

## ما هو RTL؟

### الأساسيات

RTL (من اليمين إلى اليسار) يشير إلى أنظمة الكتابة حيث يُكتب النص ويُقرأ من اليمين إلى اليسار. الخطوط RTL الرئيسية هي:

| الخط | اللغات | المتحدثون |
|------|--------|----------|
| العربي | العربية، الفارسية، الأردية، البشتونية، الكردية، الأويغورية | ~٤٠٠+ مليون |
| العبري | العبرية، اليديشية | ~٩ مليون |
| السرياني | السريانية، الآرامية | ~٤٠٠,٠٠٠ |
| الثانا | الديفيهي (المالديف) | ~٣٥٠,٠٠٠ |

### ما الذي يتدفق بالضبط من اليمين إلى اليسار؟

في خطوط RTL، التالي يتدفق من اليمين إلى اليسار:
- الحروف داخل الكلمات
- الكلمات داخل الجمل
- ترتيب القراءة عبر الأسطر
- المسح البصري العام

ومع ذلك، قد تبقى بعض العناصر LTR:
- النص اللاتيني أو الأرقام المضمنة (أحياناً)
- التدوين الرياضي
- كود البرمجة

هذا الخلط يسمى **النص ثنائي الاتجاه** أو "BiDi"، وله قواعده المعقدة الخاصة.

## RTL مقابل LTR: الفروقات الرئيسية

### التخطيط البصري

عندما تنعكس صفحة لـ RTL:

```
تخطيط LTR:                    تخطيط RTL:
┌─────────────────────┐       ┌─────────────────────┐
│ Logo    [Nav] [Nav] │       │ [Nav] [Nav]    Logo │
├─────────────────────┤       ├─────────────────────┤
│ شريط    │  المحتوى  │       │  المحتوى  │ شريط    │
│ جانبي   │           │       │           │ جانبي   │
└─────────────────────┘       └─────────────────────┘
```

### ما يجب أن ينعكس

العناصر التي تنعكس عادة في RTL:
- ترتيب التنقل
- مواقع الأشرطة الجانبية
- الأيقونات التي تشير إلى الاتجاه (الأسهم، المثلثات)
- مؤشرات التقدم
- اتجاهات السحب
- محاذاة النص (المحاذاة لليسار تصبح لليمين)
- الهامش والحشو (اليسار يصبح يمين)

### ما لا يجب أن ينعكس

بعض العناصر يجب أن تبقى دون تغيير:
- **الأيقونات**: أزرار التشغيل، علامات الصح، معظم الأيقونات غير الاتجاهية
- **الصور**: الصور، الرسوم التوضيحية (ما لم تكن غير مناسبة ثقافياً)
- **أرقام الهاتف**: تُعرض دائماً بالترتيب القياسي
- **عناصر التحكم في الفيديو**: التشغيل/الإيقاف تبقى قياسية
- **الخرائط**: التوجيه الجغرافي يبقى كما هو

## تنفيذ RTL في HTML/CSS

### سمة dir

أساس دعم RTL هو سمة HTML `dir`:

```html
<!-- تعيين الاتجاه على مستوى المستند -->
<html lang="ar" dir="rtl">

<!-- أو على عناصر محددة -->
<div dir="rtl">
  محتوى باللغة العربية
</div>
```

### سمة lang

قم دائماً بإقران `dir` مع سمة `lang` المناسبة:

```html
<html lang="ar" dir="rtl">  <!-- العربية -->
<html lang="fa" dir="rtl">  <!-- الفارسية -->
<html lang="he" dir="rtl">  <!-- العبرية -->
<html lang="ur" dir="rtl">  <!-- الأردية -->
```

### خصائص CSS المنطقية

يقدم CSS الحديث **خصائص منطقية** تتكيف تلقائياً مع اتجاه الكتابة:

```css
/* خصائص فيزيائية (لا تتكيف) */
.طريقة-قديمة {
  margin-left: 20px;
  padding-right: 10px;
  text-align: left;
}

/* خصائص منطقية (تتكيف مع الاتجاه) */
.طريقة-جديدة {
  margin-inline-start: 20px;
  padding-inline-end: 10px;
  text-align: start;
}
```

### Flexbox و Grid

يدعم Flexbox و CSS Grid بشكل طبيعي RTL:

```css
.container {
  display: flex;
  flex-direction: row; /* ينعكس تلقائياً في RTL */
}
```

لا حاجة لـ CSS إضافي — فقط عيّن `dir="rtl"` على الحاوية.

## أنماط RTL الشائعة

### قوائم التنقل

```html
<!-- القائمة تنعكس تلقائياً في RTL -->
<nav dir="rtl">
  <ul class="flex gap-4">
    <li><a href="/">الرئيسية</a></li>
    <li><a href="/about">حول</a></li>
    <li><a href="/contact">اتصل بنا</a></li>
  </ul>
</nav>
```

### تخطيطات النماذج

```html
<form dir="rtl">
  <div class="form-group">
    <label for="name">الاسم</label>
    <input type="text" id="name" />
  </div>
  <div class="form-group">
    <label for="email">البريد الإلكتروني</label>
    <input type="email" id="email" dir="ltr" />
  </div>
</form>
```

ملاحظة: حقول البريد الإلكتروني غالباً ما تحتاج `dir="ltr"` صريح لأن عناوين البريد الإلكتروني دائماً LTR.

## النقاط الرئيسية

١. **خطط لـ RTL مبكراً**: تعديل دعم RTL لاحقاً أصعب بكثير من بنائه من البداية.

٢. **استخدم الخصائص المنطقية**: خصائص CSS المنطقية (`inline-start`، `inline-end`) تتكيف تلقائياً مع اتجاه الكتابة.

٣. **استفد من Flexbox/Grid**: أنظمة التخطيط الحديثة تتعامل مع تغييرات الاتجاه تلقائياً.

٤. **ليس كل شيء ينعكس**: بعض العناصر (الصور، الفيديوهات، بعض الأيقونات) لا يجب أن تنقلب في RTL.

٥. **اختبر مع مستخدمين حقيقيين**: المتحدثون الأصليون بـ RTL سيلتقطون مشاكل يفوتها الاختبار الآلي.
