## تعارف

کیا ہوتا ہے جب عربی متن میں انگریزی الفاظ ہوں؟ یا جب عبرانی جملے میں نمبر شامل ہوں؟ جب بائیں سے دائیں اور دائیں سے بائیں متن ایک ساتھ ظاہر ہوں، معاملات تیزی سے پیچیدہ ہو جاتے ہیں۔

اس مخلوط متن پر غور کریں:
```
The word "مرحبا" means hello.
```

عربی لفظ "مرحبا" کو دائیں سے بائیں رینڈر ہونا چاہیے، جبکہ ارد گرد کی انگریزی بائیں سے دائیں ہے۔ کمپیوٹر کیسے جانتا ہے کہ ہر حرف کس سمت بہنا چاہیے؟

جواب ہے **یونیکوڈ دو طرفہ الگورتھم** (BiDi) — قواعد کا ایک نفیس مجموعہ جو مخلوط سمت والے متن کی ڈسپلے کو کنٹرول کرتا ہے۔ کثیر لسانی متن کے ساتھ کام کرنے والے ہر ڈویلپر کے لیے BiDi کو سمجھنا ضروری ہے۔

## دو طرفہ متن کیا ہے؟

### مسئلہ

تحریری نظاموں کی مختلف متعلقہ سمتیں ہوتی ہیں:
- **LTR (بائیں سے دائیں)**: لاطینی، یونانی، سیریلک، تھائی، وغیرہ
- **RTL (دائیں سے بائیں)**: عربی، عبرانی، سریانی، ثانا

جب مختلف سمتی نظاموں کا متن ایک ساتھ ظاہر ہو، رینڈرنگ انجن کو تعین کرنا ہوگا:
۱. پیراگراف کی مجموعی سمت
۲. پیراگراف کے اندر ہر حصے کی سمت
۳. غیر جانبدار حروف (خالی جگہیں، علامات، نمبر) کو کیسے ہینڈل کریں

### ایک سادہ مثال

انگریزی برانڈ نام کے ساتھ عربی جملہ دیکھیں:

```
دائیں سے بائیں بنیادی سمت:
← أنا أستخدم Microsoft Word يومياً ←
```

یہ جملہ اس طرح پڑھا جانا چاہیے:
۱. "أنا أستخدم" - RTL
۲. "Microsoft Word" - LTR (سرایت شدہ)
۳. "يومياً" - RTL

چیلنج یہ ہے کہ اسے صحیح طریقے سے ڈسپلے کریں جبکہ میموری میں حروف کی منطقی ترتیب برقرار رہے۔

## یونیکوڈ دو طرفہ الگورتھم

### جائزہ

یونیکوڈ BiDi الگورتھم (UBA)، **یونیکوڈ معیاری ضمیمہ #9** میں بیان کیا گیا ہے، بالکل وضاحت کرتا ہے کہ متن کی سمت کا تعین اور رینڈرنگ کیسے کی جائے۔ یہ ہر جدید براؤزر، آپریٹنگ سسٹم، اور متن رینڈرنگ انجن میں نافذ ہے۔

## عملی مضمرات

### HTML کنٹرولز

یہ خصوصیات اور حروف استعمال کریں:

```html
<!-- بنیادی سمت مقرر کریں -->
<p dir="rtl">عربی پیراگراف یہاں انگریزی کے ساتھ</p>

<!-- سرایت شدہ متن کو الگ تھلگ کریں -->
<p>لفظ <bdi>مرحبا</bdi> کا مطلب ہیلو ہے۔</p>

<!-- الگورتھم کو اوور رائیڈ کریں -->
<span dir="ltr">LTR سمت کو مجبور کریں</span>
```

### یونیکوڈ کنٹرول حروف

BiDi رویے کو کنٹرول کرنے کے لیے یہ حروف داخل کریں:

```javascript
const LRM = '\u200E';  // بائیں سے دائیں مارک
const RLM = '\u200F';  // دائیں سے بائیں مارک
const LRI = '\u2066';  // بائیں سے دائیں آئسولیٹ
const RLI = '\u2067';  // دائیں سے بائیں آئسولیٹ
const FSI = '\u2068';  // پہلا مضبوط آئسولیٹ
const PDI = '\u2069';  // پاپ ڈائریکشنل آئسولیٹ

// مثال: RTL سیاق میں انگریزی کو یقینی بنائیں
const text = `مرحباً ${LRI}Microsoft${PDI} العالم`;
```

### عام استعمال کے معاملات

#### RTL سیاق میں LTR متن داخل کرنا

```html
<!-- پروڈکٹ نام LTR رہنا چاہیے -->
<p dir="rtl">
  أنا أستخدم <span dir="ltr">iPhone 15 Pro</span> كل يوم
</p>
```

#### ای میل ایڈریس

ای میل ایڈریس ہمیشہ LTR ہونے چاہئیں:
```html
<p dir="rtl">
  البريد الإلكتروني: <bdi dir="ltr">user@example.com</bdi>
</p>
```

## عام خرابیاں

### علامات کے مسائل

علامات "غیر جانبدار" ہیں، سیاق سے سمت لیتی ہیں:

```
مسئلہ: "Hello, world" RTL سیاق میں
غلط:   "Hello, world"  ← علامات ہل جاتی ہیں
صحیح:  "Hello, world"  ← مناسب آئسولیشن کے ساتھ
```

حل: `<bdi>` یا آئسولیٹ کنٹرولز استعمال کریں۔

## اہم نکات

۱. **BiDi خودکار ہے لیکن نامکمل**: یونیکوڈ BiDi الگورتھم زیادہ تر معاملات کو ہینڈل کرتا ہے لیکن کناروں کے معاملات میں مدد کی ضرورت ہے۔

۲. **HTML کنٹرولز استعمال کریں**: `dir` خصوصیت اور `<bdi>` عنصر معنوی، قابل رسائی حل فراہم کرتے ہیں۔

۳. **آئسولیٹ کریں، اوور رائیڈ نہ کریں**: `<bdi>` اور آئسولیٹ کنٹرولز ایمبیڈنگز یا اوور رائیڈز سے زیادہ محفوظ ہیں۔

۴. **حقیقی مواد کے ساتھ ٹیسٹ کریں**: مصنوعی ٹیسٹ حقیقی دنیا کی پیچیدگی کو چھوڑ دیتے ہیں۔

۵. **غیر جانبدار حروف کو سیاق کی ضرورت ہے**: علامات، خالی جگہیں، اور نمبر آس پاس کے متن کی سمت کی بنیاد پر برتاؤ کرتے ہیں۔
